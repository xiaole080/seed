# 7日パイロット 配布ガイド

実ユーザー 1〜2名に7日間使ってもらい、毎日5分のヒアリングで気づきを集めるためのドキュメント。

## 0. 配布の全体像

1. **公開デプロイ** → `https://seed-XXXXX.pages.dev/` のような恒久 URL を取得（PC 起動不要）
2. **テスター招待** → LINE / メールで URL + 簡単な説明 + 同意を送る
3. **PWA インストール** → スマホで URL を開き「ホーム画面に追加」
4. **7日間運用** → 毎日5分ヒアリング、Google Sheets を見ながら傾向確認
5. **撤収** → 7日目に振り返り、データ取り扱いを伝える

---

## 1. デプロイ (Cloudflare Pages — 推奨)

無料・常時稼働・カード登録なし・Vite を自動検出。所要約20分。

### 手順

1. **GitHub にコードを push**
   - 既存リポジトリがあるならそこへ。無ければ `gh repo create seed --private --source=. --push`
   - `.env.local` と `node_modules` は `.gitignore` で除外済み ✓
   - `.design/` も `.gitignore` 済み ✓

2. **Cloudflare アカウント作成** → https://dash.cloudflare.com/sign-up （無料、メールのみ）

3. **Pages プロジェクト作成**
   - ダッシュボード → Workers & Pages → Create application → Pages → Connect to Git
   - GitHub アカウント連携 → `seed` リポジトリ選択
   - Build settings:
     - Framework preset: **Vite**
     - Build command: `npm run build`
     - Build output directory: `dist`
   - Environment variables:
     - `VITE_SHEETS_ENDPOINT` = （`.env.local` に貼ったのと同じ GAS URL）

4. **Save and Deploy** → 1〜2分待つと URL が発行される: `https://seed-XXXXX.pages.dev/`

5. **動作確認**: その URL をスマホで開いて、気分を1件記録 → Google Sheets に行が増えれば成功

### 代替案

- **Vercel** も同等。GitHub 連携 → Import → 環境変数設定 → Deploy。CF Pages とほぼ同じ手順
- **Netlify Drop**: `npm run build` した `dist/` をブラウザにドラッグ&ドロップで一発公開（env var は別途設定）

### 注意点

- `VITE_SHEETS_ENDPOINT` はビルド時に JS に**埋め込まれる**ので、公開後はビルド成果物を見れば誰でも GAS URL を取れる。低リスクだが、悪意ある書き込みが来たら `gas/Code.gs` の `SHARED_SECRET` を設定して両側に揃える
- カスタムドメインを当てたい場合は CF Pages → Custom domains から、ドメイン無くても `*.pages.dev` で十分

LINE / メールで送る。**敬語で簡潔・任意性・撤収方法を必ず含める。**

```

【何をするアプリ?】
```

## 3. 同意・プライバシー (必須)

就労移行支援の利用者を対象にする場合は、**書面または LINE 上で明文同意** を得てから配布する。

### 最小限の同意事項

- [ ] アプリの目的（自分の体調を可視化する、開発者の改善学習のため）
- [ ] 収集する情報（気分1〜5、睡眠/食事/服薬/運動/体調タグ、ひとこと、通所打刻、ニックネーム、デバイスID）
- [ ] 保存先（開発者個人の Google スプレッドシート）
- [ ] 共有しない（第三者・SNS・支援機関いずれにも）
- [ ] 撤収権（いつでも削除できる、削除の手順）
- [ ] つらいときの相談先がアプリ内にある旨
- [ ] 連絡先（質問・撤収依頼の窓口）

### 倫理的配慮（特に支援センター経由で募集する場合）

- 支援員に先に同意を取り、利用者への声掛けは支援員から行う
- 「使わないと不利になる」と感じさせない（参加任意・撤収自由を強調）
- 未成年や成年後見人がいる利用者の場合は保護者・後見人同意
- 危機状態（希死念慮、急性症状）にあると判断される方には今回パイロットから外す

## 4. 毎日5分ヒアリング (テンプレ)

毎日同じ質問で取り、傾向の変化を見る。LINE/対面/録音、いずれの形式でも OK。

### 共通6問（毎日同じ）

1. 今日 開いた回数 / 開けた時間帯
2. 記録してて 「面倒だな」 と思った瞬間あった?
3. 鳥のささやき (画面の文字)、目に入った? 残ったコトバある?
4. 「あれ?」「困った」 と感じたところ
5. 今日のアプリ全体 1〜5点 (1=つらかった / 5=よかった)
6. 明日も使う気ある? (はい/いいえ/わからない)

### 7日目だけ追加

7. 7日続けて、何か気づいたこと
8. 続けたい? やめたい? どんな機能なら続けやすそう?
9. データを開発者の Sheets に残しておいて OK? 削除したい?

### 観察ログ Sheet テンプレ (任意)

Google スプレッドシートを1枚追加して以下のヘッダで:

```
date | tester | open_count | annoyance | whisper_recall | issue | rating_1_5 | will_use_tomorrow | notes
```

Sheets 側で既に `mood_logs` `attendance` がたまっているので、ヒアリングログと突き合わせて分析できる。

## 5. 配布前チェックリスト

公開デプロイ後、配る前に必ず通す:

- [ ] 公開 URL をスマホで開けて、気分1件記録 → `mood_logs` に行が増える
- [ ] スマホで 「ホーム画面に追加」 が動く、アイコンが出る
- [ ] 「じぶん」 タブに 「つらいときの相談先」 カードがある、電話番号タップで `tel:` が起動する
- [ ] ブラウザリロード でログイン画面に戻らない (state 永続化)
- [ ] DevTools コンソール (PC) で `[sheets] post failed` が出ていない
- [ ] テスター用のメッセージ・同意 文面を準備済み
- [ ] ヒアリング日時を決めて (毎日固定時刻が理想) テスターと共有
- [ ] 観察ログ Sheet を用意

---

## 6. North Star 指標 (PM 推奨)

**Low-day Return Rate** = 気分3以下を記録した翌日にもアプリを開いた率

7日終わったら Sheets で集計:
- `mood_logs` で `mood <= 3` の日の翌日 (24時間以内に再ログイン) があったか
- N=テスター数×日数 で割合を出す
- 目標: 70%

これを「DAUいくつ」「streakいくつ」より優先する。**しんどい日の翌日に戻れるアプリかどうか**こそがこのプロダクトの存在意義。

---

## 7. やってはいけない（パイロット中）

- 機能追加。バグ修正以外は7日間我慢
- テスターに「もっと使って」と促す（バイアス入る）
- 自分のニックネームでテスト用に記録する（実データに混ざる）
- テスター同士でデータを見せ合う / Sheets を共有する
- 7日中の解約防止のため UI で粘る（責めない設計を破る）
