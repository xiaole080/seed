# プロジェクト方針

このリポジトリは、気分・睡眠・服薬・通所記録・自由記述を扱うローカルファーストの体調記録アプリです。
機微な健康データを扱うため、開発フローとプライバシーの扱いに以下のルールを必ず守ること。

## 開発ワークフロー（必須の順序）

機能追加・仕様変更・UI修正の依頼を受けたら、原則として次の順序で subagent に作業を割り当てること。
途中のステップを飛ばさない。

1. **product-manager** — 仕様を読み、タスクに分解し、受け入れ条件を書く。MVP範囲外の要素はここで止める。
2. **frontend-engineer** — 受け入れ条件に沿って実装する。最小差分で行う。
3. **qa-tester** — lint / build / test を実行し、未入力・任意入力・自由記述などのエッジケースを確認する。
4. **privacy-reviewer** — 個人データの取り扱いを監査する。コードは変更せず指摘のみ。

軽微な修正（タイポ、コメント、スタイル微調整など、データの保存先・送信・記録項目に一切影響しないもの）は、
このフローを省略してよい。判断に迷う場合はフローを通すこと。

## privacy-reviewer を必ず通すケース（省略禁止）

以下のいずれかに該当する変更は、規模が小さくても **必ず privacy-reviewer のレビューを最後に通す**こと。

- 外部送信に関わるコードの追加・変更（fetch / axios / XHR / sendBeacon / 外部スクリプト / 解析・監視SDK）
- データの保存先に関わる変更（localStorage / Cookie / サーバー / サードパーティ）
- 記録項目の追加・変更（特に自由記述・気分・睡眠・服薬・通所記録）
- Google Sheets エクスポート / CSV 出力の項目や挙動の変更
- 同意取得・削除導線に関わる変更

## データ取り扱いの基本原則

- 健康データの保存先は localStorage を原則とする。
- 自由記述などの機微な内容を、ユーザーの明示操作なしに外部へ送らない。
- 外部送信（Google Sheets 等）は任意・オプトインとし、送信前にユーザーが内容を確認できるようにする。
- API キー・シークレットをコードにハードコードしない。
- ユーザーが自分の記録を削除できる導線を維持する。

## 各 subagent の定義場所

`.claude/agents/` 配下に以下を配置している。
- product-manager.md
- frontend-engineer.md
- qa-tester.md
