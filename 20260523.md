# Seedアプリ 修正仕様書：ケア・じぶん・ホーム・きろく・気分記録画面の統合修正

## 1. 依頼

Seedアプリについて、以下のバグ修正・仕様変更を実装してください。

今回は新しい大規模機能ではなく、既存の以下の画面の動作整合性を直すことが目的です。

- ケア
- じぶん
- ホーム
- きろく
- 気分を記録する

特に重要なのは、以下です。

1. ケアページの目標管理を正しくする
2. じぶん画面の記録項目ON/OFFを、きろく画面・気分記録画面へ反映する
3. ホームから「今日の様子」と「昨日の様子」を記録・修正できるようにする
4. きろく画面を、日付ありの時系列表示にする
5. 記録がない日も空きとして表示する
6. 各選択肢に必要に応じて「その他」を追加する
7. 自由記述・その他欄は外部送信しない

---

## 2. 基本設計方針

### 2.1 記録は「入力した日時」ではなく「対象日」で保存する

記録は、ユーザーが入力した時刻ではなく、記録対象日ごとに保存してください。

例：

- 今日の様子を記録する  
  → 今日の日付の `DailyRecord` に保存

- 昨日の様子を記録する  
  → 昨日の日付の `DailyRecord` に保存

- 今日の記録を修正する  
  → 今日の日付の `DailyRecord` を上書き

- 昨日の記録を修正する  
  → 昨日の日付の `DailyRecord` を上書き

朝に通所してから昨日分を記録する利用者も多いため、**昨日の食事・運動・体調・服薬・気分も記録できる**ようにしてください。

---

## 3. localStorageキー

既存のlocalStorageキーを確認し、できるだけ既存キーを使ってください。

想定キー：

- `seed.daily.v1`
- `seed.attendance.v1`
- `seed.app.state.v1`
- `seed.care.goals.v1`
- `seed.consent.v1`

存在しない場合は、用途が分かる名前で追加してください。

ただし、同じ意味のデータを複数キーに分散させないでください。

---

## 4. DailyRecordの保存仕様

### 4.1 基本形

`DailyRecord` は日付単位で保存してください。

```ts
type DailyRecord = {
  date: string; // YYYY-MM-DD

  moodScore?: number;
  influences?: string[];
  influenceOtherText?: string;

  details?: {
    sleep?: SleepRecord;
    meal?: MealRecord;
    exercise?: ExerciseRecord;
    condition?: ConditionRecord;
    medication?: MedicationRecord;
  };

  notes?: {
    privateNote?: string;
  };

  createdAt?: string;
  updatedAt: string;
  edited?: boolean;
};
```

### 4.2 今日・昨日の保存先

今日が `2026-05-23` の場合：

```txt
今日の様子を記録する
→ date: "2026-05-23"

昨日の様子を記録する
→ date: "2026-05-22"
```

同じ日付の記録がすでに存在する場合は、新規作成せず、同じ `date` のレコードを上書き更新してください。

### 4.3 旧DailyRecordとの互換性

既存の `seed.daily.v1` のデータを壊さないでください。

要件：

- 旧形式の `DailyRecord` を読み込めるようにする
- 旧形式の `moodScore`
- 旧形式の `primaryInfluence`
- 旧形式の `primaryInfluences`
- 旧形式の `details`
- 旧形式の `note`

がある場合は、新形式の今日・対象日データとして扱ってください。

新しく保存する時は新形式で保存してよいですが、既存データが消えないようにしてください。

## 5. ケアページ修正

### 5.1 小さな目標を毎日リセットする

#### 現状

ケアページの「小さな目標」が翌日になっても残ってしまう、または日付単位で管理されていない可能性があります。

#### 修正内容

小さな目標は「その日だけの目標」として扱ってください。

要件：

- 今日の小さな目標は、日付ごとに保存する
- 日付が変わったら、前日の小さな目標は今日の目標として表示しない
- 前日の目標を履歴として保持するかどうかは、既存設計に合わせる
- 少なくとも、翌日にホーム/ケアページを開いた時、昨日の目標がそのまま今日の目標として表示されないようにする

保存例：

```ts
type SmallGoal = {
  id: string;
  date: string; // YYYY-MM-DD
  text: string;
  done?: boolean;
  createdAt: string;
  updatedAt?: string;
};
```

---

### 5.2 「気になる目標」が追加できないバグ修正

#### 現状

ケアページで「気になる目標」を追加しようとしても、追加できないバグがあります。

#### 修正内容

以下を確認して修正してください。

確認項目：

- 入力欄の state が正しく更新されているか
- 追加ボタンの `onClick` が発火しているか
- 空文字判定で弾かれすぎていないか
- localStorage または state に保存されているか
- 追加後に画面へ即時反映されているか
- ページ再読み込み後も残るか

要件：

- 入力 → 追加 → 一覧に表示
- 空文字は追加しない
- 追加後は入力欄をクリア
- 追加した目標は削除できるようにする
- 既存UIの雰囲気はなるべく維持する

---

## 6. じぶん画面修正：記録項目ON/OFFの反映

### 6.1 現状

じぶん画面で記録項目を追加・ON/OFFしても、きろく画面の表示内容や、気分を記録する画面の表示項目が変わっていません。

### 6.2 修正内容

じぶん画面の「記録する項目」のON/OFFを、以下の画面へ即時反映してください。

反映先：

- きろく画面
- ホーム → 今日の様子を記録する画面
- ホーム → 昨日の様子を記録する画面
- 今日の記録修正画面
- 昨日の記録修正画面

対象項目例：

- 睡眠
- 食事
- 運動
- 体調
- 服薬
- その他、既存実装にある記録項目

### 6.3 要件

- ONの項目だけ、気分記録画面に表示する
- OFFの項目は、気分記録画面に表示しない
- ONの項目だけ、きろく画面のカードや可視化に表示する
- OFFにしても過去データは削除しない
- OFFにした項目は「新規記録画面では非表示」にするだけ
- 再度ONにしたら、過去データも含めてきろく画面で表示可能にする
- localStorageの設定変更後、ページリロードなしでUIに反映されるようにする

実装時は、記録項目設定を1つの共通state / hook / utilityから参照するようにしてください。

画面ごとに別々の設定を持たないでください。

## 7. ホーム画面修正：今日・昨日の記録導線

### 7.1 背景

利用者は夜に記録するとは限らず、朝の通所時に前日の状態を振り返って記録する人も多いです。

そのため、ホーム画面から以下の2種類の記録導線を選べるようにしてください。

- 今日の様子を記録する
- 昨日の様子を記録する

また、対象日の記録がすでにある場合は、新規記録ではなく修正導線にしてください。

---

### 7.2 ホーム画面のボタン表示

#### 今日の記録がない場合

ボタン文言：

```txt
今日の様子を記録する
```

押下時：

- 今日の日付を対象日として、新規記録画面へ遷移

#### 今日の記録がある場合

ボタン文言：

```txt
今日の記録を修正する
```

押下時に確認ダイアログを表示：

```txt
今日はすでに記録済みです。内容を修正しますか？
```

選択肢：

- 修正する
- キャンセル

「修正する」を選んだ場合：

- 今日の日付の `DailyRecord` を読み込む
- 前回の記録内容が入った状態で記録画面を開く
- 保存時は今日の日付の `DailyRecord` を上書きする
- 新しい別レコードを作らない

---

#### 昨日の記録がない場合

ボタン文言：

```txt
昨日の様子を記録する
```

押下時：

- 昨日の日付を対象日として、新規記録画面へ遷移

#### 昨日の記録がある場合

ボタン文言：

```txt
昨日の記録を修正する
```

押下時に確認ダイアログを表示：

```txt
昨日の記録を修正しますか？
```

選択肢：

- 修正する
- キャンセル

「修正する」を選んだ場合：

- 昨日の日付の `DailyRecord` を読み込む
- 前回の記録内容が入った状態で記録画面を開く
- 保存時は昨日の日付の `DailyRecord` を上書きする
- 新しい別レコードを作らない

---

### 7.3 ホーム画面の構造

推奨構造：

```txt
ホーム
├ 今日の様子を記録する / 今日の記録を修正する
└ 昨日の様子を記録する / 昨日の記録を修正する
```

## 8. 気分記録画面修正

### 8.1 記録対象日に応じて画面タイトルを変える

記録対象日と新規/修正状態に応じて、画面タイトルを変えてください。

例：

- 今日の様子を記録する
- 昨日の様子を記録する
- 今日の記録を修正する
- 昨日の記録を修正する

---

### 8.2 今日分として記録できる項目

今日分の記録では、以下を入力できるようにしてください。

- 今日の気分
- 今日の影響要因
- 今日の食事
- 今日の運動
- 今日の体調
- 今日の服薬
- 今日のメモ

睡眠は、今日分の記録であっても、文言としては以下のように扱ってください。

```txt
昨夜〜今朝の睡眠
```

---

### 8.3 昨日分として記録できる項目

昨日分の記録でも、以下を入力できるようにしてください。

- 昨日の気分
- 昨日の影響要因
- 昨日の睡眠
- 昨日の食事
- 昨日の運動
- 昨日の体調
- 昨日の服薬
- 昨日のメモ

画面上では、対象日が昨日であることが分かる文言にしてください。

例：

- 昨日の気分
- 昨日の食事
- 昨日の運動
- 昨日の体調
- 昨日の服薬
- 昨日のメモ

---

### 8.4 修正時のフォーム復元

今日または昨日の記録を修正する場合、対象日の `DailyRecord` の内容をフォームに復元してください。

要件：

- 気分スコアを復元
- 影響要因を復元
- 睡眠を復元
- 食事を復元
- 運動を復元
- 体調を復元
- 服薬を復元
- メモを復元
- その他欄の入力内容を復元

保存時は、同じ `date` の `DailyRecord` を上書きしてください。

---

### 8.5 記録項目ON/OFFとの連動

気分記録画面では、じぶん画面でONになっている項目だけを表示してください。

例：

- 食事がON → 食事項目を表示
- 食事がOFF → 食事項目を非表示
- 睡眠がON → 睡眠項目を表示
- 睡眠がOFF → 睡眠項目を非表示

ただし、修正画面でOFFになっている項目に過去データが存在する場合、その過去データを消さないでください。

保存時も、非表示項目の既存データを勝手に削除しないでください。

---

## 9. 「その他」選択肢の追加

### 9.1 背景

利用者の状態や気分に影響する要因は、固定選択肢だけでは拾いきれません。

特に、選択肢にはないが本人にとって影響が大きい出来事を記録できるようにする必要があります。

---

### 9.2 対象項目

以下の選択式項目には、必要に応じて「その他」を追加してください。

対象：

- 気分への影響要因
- 睡眠で気になったこと
- 食事で気になったこと
- 運動できなかった理由
- 体調で気になること
- 服薬で気になること

---

### 9.3 UI仕様

各選択肢の最後に「その他」を置いてください。

ユーザーが「その他」を選択した場合のみ、短い自由入力欄を表示してください。

表示例：

```txt
その他
具体的に書く
```

入力欄の補足文：

```txt
個人名・施設名などを書きすぎないでください
```

---

### 9.4 保存仕様

「その他」は、選択されたことと入力内容を分けて保存してください。

例：

```ts
{
  influences: ["sleep", "task", "other"],
  influenceOtherText: "朝の電車が混んでいて疲れた"
}
```

または `details` 内に保存してください。

```ts
{
  details: {
    condition: {
      tags: ["headache", "other"],
      otherText: "気圧の変化がつらかった"
    }
  }
}
```

---

### 9.5 その他欄の扱い

自由入力は本人にとって重要な情報ですが、個人情報・健康情報・心理情報を含む可能性があります。

そのため、以下を守ってください。

- その他の自由入力はローカル保存のみ
- Google Sheetsなど外部には送信しない
- CSV出力にも原則含めない
- きろく画面では、本人が開いた時だけ見える折りたたみ表示にする
- 支援員共有用の出力には含めない
- もし支援員共有に含める場合は、必ず明示確認を挟む

---

### 9.6 バリデーション

- その他を選んでも、自由入力は空欄OK
- 空欄の場合は `otherText: ""` または `undefined` で保存
- 文字数上限は100文字程度
- 改行は不要
- 入力欄には「個人名・施設名などを書きすぎないでください」と補足を表示

---

### 9.7 表示文言

禁止：

- その他の原因
- 問題点
- 悪化要因

推奨：

- その他、気になったこと
- 選択肢にないこと
- 自分用メモ

---

## 10. きろく画面修正：日付ありの時系列表示

### 10.1 現状

きろく画面が日付ごとの時系列として見えにくい、または記録がない日が表示されていません。

---

### 10.2 修正内容

きろく画面を、日付つきの時系列表示に変更してください。

要件：

- 7日 / 14日 / 今月 の期間切り替えを維持する
- 選択期間内の日付をすべて表示する
- 記録がある日は、その日の記録内容を表示する
- 記録がない日は空欄または「記録なし」と表示する
- 記録がない日を0点扱いしない
- 気分スコアの平均を出す場合は「記録した日の平均」と表記する
- 未記録を責める表現は使わない

---

### 10.3 表示例

```txt
2026/05/22
- 気分：4
- 影響していそうなこと：睡眠、予定
- 睡眠：途中で起きた
- 通所：10:05 - 15:30
- 自分用メモあり

2026/05/21
- 記録なし

2026/05/20
- 気分：3
- 影響していそうなこと：人間関係
- 食事：少なめ
```

---

### 10.4 その他・自由記述の表示

その他の入力や自由記述がある場合、日付ごとの記録内に折りたたみで表示してください。

一覧に自由記述をそのまま露出しないでください。

表示例：

```txt
2026/05/22
- 気分：3
- 影響していそうなこと：睡眠、その他
- 自分用メモあり
```

「自分用メモあり」を押すと、その他欄やメモを表示する。

---

### 10.5 記録項目ON/OFFとの連動

きろく画面では、じぶん画面でONになっている項目を優先して表示してください。

要件：

- ONの項目はカードや時系列内に表示する
- OFFの項目は通常表示しない
- ただし、過去データは削除しない
- 再度ONにした場合、過去データも表示できる
- OFF項目に過去データがある場合は、必要なら「非表示中の記録あり」のような控えめな表示にしてもよい

---

## 11. 通所記録との連動

きろく画面の日付ごとの時系列には、可能であれば通所記録も表示してください。

使用データ：

- `seed.attendance.v1`

表示内容：

- 通所予定
- 実際の check-in
- 実際の check-out
- 通所済み / 在宅 / 休み / 未打刻

注意：

- 未打刻を「欠席」と断定しない
- 表示は「未打刻」または「記録なし」にする

## 12. UXルール

### 12.1 未記録を責めない

以下の表現は禁止です。

- 記録できていません
- 継続できていません
- さぼっています
- 改善が必要です
- 悪化しています

代わりに以下の表現を使ってください。

- 記録なし
- 記録がある日をもとに表示しています
- 無理に全部記録しなくて大丈夫です
- 必要なときだけ振り返れます

---

### 12.2 医療診断に見せない

以下の表現は禁止です。

- うつ傾向
- 双極傾向
- 症状が悪化
- 治療が必要
- 再発リスク
- 危険状態

代わりに以下の表現を使ってください。

- 気分に波があります
- 少し疲れが出やすい日がありそうです
- 睡眠と気分が関係している可能性があります
- 支援員さんと一緒に振り返る材料になります

---

### 12.3 記録の哲学

このアプリでは、記録を「できた / できない」で評価しないでください。

基本方針：

- 記録しない日があってもよい
- 記録なしは、欠席・不調・悪化を意味しない
- 未入力は0点ではない
- 本人が振り返りたい時に使えることを重視する
- 支援員との対話材料にする
- 医療診断の代替にしない

---

## 17. 補足：設計上の最終判断

この統合版では、以前の案である **「昨日の様子 / 今日の様子を1つのDailyRecord内で分ける設計」** は採用しません。

理由は、朝に通所してから昨日分を記録する利用者が多いためです。

そのため、最終設計は以下に統一します。

```txt
ホーム
├ 今日の様子を記録する / 今日の記録を修正する
└ 昨日の様子を記録する / 昨日の記録を修正する
```

保存単位は、入力日時ではなく **対象日ごとのDailyRecord** です。

```txt
今日の様子を記録する
→ 今日の日付のDailyRecordへ保存

昨日の様子を記録する
→ 昨日の日付のDailyRecordへ保存
```

この方針により、きろく画面では日付順に自然に表示できます。
