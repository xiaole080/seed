# 5 人テスト向け手引き

Seed 0.1.0 のテストにご協力いただきありがとうございます。
本文書は **テスト参加者** と **配布者** が両方読むことを想定しています。

## 1. 起動（ローカル開発）

開発環境で動かす場合は次の通り。

```bash
# 1. 依存をインストール
npm install

# 2. 開発サーバを起動 (http://localhost:5173 が開く)
npm run dev

# 3. 本番ビルド
npm run build

# 4. テスト一式
npm test
```

Node.js 18 以上、npm 9 以上。

## 2. テスト URL（GitHub Pages）

配布者は GitHub Pages にデプロイされた URL を参加者に共有してください。

```
https://<your-github-account>.github.io/seed/
```

> 本リポジトリはまだデプロイされていません。下記「デプロイ手順」を参照。

## 3. testerId の入力ガイド

最初の同意画面のあと、ログイン画面でニックネームを入力します。
このニックネームがそのまま testerId として扱われます。

- **推奨例**: `tester-01`, `tester-02`, ..., `tester-05`
- 自分のあだ名でも構いません（例: `あおむぎ`）
- ただし、本名や、家族・支援員の名前は使わないでください
- アプリ内とテスト記録に表示されます

データ分離は行わないため、1 端末 1 人での利用を前提とします。
複数 testerId で同じ端末を共有しないでください（記録が混ざります）。

## 4. スマホでの確認手順

PC ブラウザで開いた後、Chrome DevTools のモバイルエミュレーションで以下 3 サイズを確認することを推奨します。

| 端末名         | width × height |
| -------------- | -------------- |
| iPhone SE      | 375 × 667      |
| iPhone 12 Pro  | 390 × 844      |
| Pixel 7 / 5    | 412 × 915      |

詳細手順は [`docs/mobile-check/README.md`](docs/mobile-check/README.md) を参照。

実機（iPhone / Android）から GitHub Pages の URL を直接開いて利用するのが最も実環境に近いです。

## 5. このバージョンで試してほしいこと

- ホームから「今日の様子を記録する」「昨日の様子を記録する」
- 既に記録済みの日に再度入ると「修正しますか？」の確認が出るか
- 「じぶん」画面の記録項目 ON/OFF が、気分記録画面・きろく画面に反映されるか
- 気分記録の自由記述（メモ）が、書いたあとに自分の端末にだけ残ること
  - スマホを閉じてから再度開いても、入力した内容が消えていないか
- 「じぶん」画面の「データを書き出す（JSON）」で JSON ファイルが端末に保存されるか
- 「じぶん」画面の「データを消す」で確認のあと、すべての記録が消えるか

## 6. 既知の未対応・既知の制限

- **複数 testerId のデータ分離は次回以降の対応です。** 同じ端末で複数 testerId を切り替えると記録が混ざります。
- スクリーンショット保存・PDF 出力はありません。
- 通所 CSV の月次集計は実打刻ベースで、未打刻日は空欄として出力されます。
- 気分記録の Google Sheets 連携は配布者側の設定が必要です（`.env.local` の `VITE_SHEETS_ENDPOINT`）。未設定なら no-op で動作します。
- 研究目的での利用には**別途同意・倫理審査**が必要です。本 5 人テストはセルフケアアプリの使い勝手の確認が主目的です。

- 報告先（プレースホルダ）: `<配布者の連絡先>`

## デプロイ手順（配布者向け）

本リポジトリはまだ GitHub に push されていません。次の流れで初回デプロイができます。

```bash
# 1. GitHub に空のリポジトリ `seed` を作成しておく (Web から)

# 2. リポジトリ初期化
git init
git add .
git commit -m "Initial commit"
git branch -M main

# 3. リモートを設定して push
git remote add origin git@github.com:<your-account>/seed.git
git push -u origin main
```

GitHub の Web 画面から:

1. リポジトリの **Settings → Pages** へ
2. **Source** を **GitHub Actions** に変更
3. main 向けの push があると `.github/workflows/deploy.yml` が動き、`dist/` を Pages にデプロイする
4. ワークフロー完了後、`https://<account>.github.io/seed/` が利用可能になる

`vite.config.ts` で `base: '/seed/'` を本番ビルドのみ有効にしているため、リポジトリ名は **必ず `seed`** にしてください。
別名にする場合は `vite.config.ts` の `base` を合わせて変更します。
